# 内容可读性规则

适用于对话回答、自动化结果、市场数据分析、风险说明和数据观察等需要用户认真阅读的内容区域。

- 内容正文不得使用过小字号。桌面端密集布局下，分析正文以 `12px` 为最低基线，并保持不低于 `1.55` 的行高。
- 模块标题不得小于正文；状态标签、数量和操作按钮等次要信息可以更小，但不低于 `10px`。
- `8px`、`9px` 只允许用于不承载决策信息的极次要元数据，不能用于结论、风险、证据、条件或操作建议。
- 结构化分析结果默认全部展开，确保用户第一眼能看到完整结论。长模块可以保留手动折叠能力，但不能默认隐藏。
- 每次调整内容型界面后，至少检查一个桌面宽窗口和一个窄窗口，确认没有文字溢出、卡片挤压或按钮重叠。
