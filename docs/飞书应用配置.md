# 飞书应用配置

韭菜盒子支持两类飞书能力：

- 主动提醒：把买卖机会、风险和任务故障推送给指定个人或群聊。
- 双向对话：私聊机器人可直接提问；群聊中只有 `@机器人` 后才会回答。

机器人只做分析和提醒，不会下单、撤单或修改交易记录。

## 1. 创建应用并启用机器人

1. 打开[飞书开发者后台](https://open.feishu.cn/app)，创建或选择一个企业自建应用。
2. 在「应用能力」中启用机器人。
3. 在「凭证与基础信息」中取得 App ID 和 App Secret。

首次在运行韭菜盒子的电脑上配置凭证时，由运维执行：

```bash
lark-cli config init --new
```

## 2. 申请权限

在「权限管理」中申请以下权限：

| 权限 | 用途 |
| --- | --- |
| `im:message.p2p_msg:readonly` | 接收用户发给机器人的私聊消息 |
| `im:message.group_at_msg:readonly` | 接收群聊中 @机器人的消息 |
| `im:message:send_as_bot` | 以机器人身份回复消息 |
| `im:chat:read` | 在韭菜盒子中搜索机器人可见的群聊 |

不需要申请读取全部群消息的权限。

## 3. 配置长连接事件

1. 进入「事件与回调 → 事件配置」。
2. 选择「使用长连接接收事件」。
3. 添加「接收消息」事件：`im.message.receive_v1`。

长连接不需要配置公网回调地址，但运行韭菜盒子的电脑需要保持联网和应用在线。详细说明见[飞书长连接官方文档](https://open.feishu.cn/document/server-docs/event-subscription-guide/event-subscription-configure-/request-url-configuration-case)。

## 4. 设置可用范围并发布

1. 把需要私聊机器人的成员加入应用可用范围。
2. 把机器人加入需要接收提醒或对话的群聊。
3. 创建应用版本并发布；企业开启审核时，需要管理员审批。

机器人能力、权限或事件订阅发生变化后，都要重新发布版本才会生效。

## 5. 在韭菜盒子中检查

打开「设置 → 飞书提醒」：

1. 连接当前飞书账号，或搜索并选择提醒群聊。
2. 查看「飞书双向对话」是否显示运行中。
3. 异常时按页面提示检查权限、事件和应用版本，然后点击「重新连接」。

请直接私聊机器人提问；在群聊中使用 `@机器人 你的问题`。普通群消息不会触发回答。
