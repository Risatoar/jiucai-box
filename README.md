# 韭菜盒子

韭菜盒子是一款基于 Trade Master 的本地优先交易辅助桌面应用。它覆盖盘前选股、盘中监控、盘后复盘和策略进化，并把 AI 对话、持仓、关注池、定时任务、飞书通知与 macOS 菜单栏整合在同一个 Codex 式工作台中。

> 产品只做行情整理、交易分析、风险控制和只读提醒；不会连接、预填或操作真实/模拟券商，也不承诺收益。

## 当前能力

- 首次初始化：核心依赖与事实仓自动准备，并展示阶段进度。
- 交易画像：收集资金、风险回撤、交易周期、品种、习惯和收益目标；生成可解释的风格评级，可在设置中二次编辑。
- AI 会话：历史会话、连续对话、持仓核对、标的分析和行动卡；最近对话与定时任务会话独立分组并支持展开收起，本机 Codex 与兼容 API 均支持实时运行阶段和流式文字展示。
- 跨对话记忆：参考 Codex 的“使用 / 生成”双开关，相关长期记忆会注入新回答；支持单会话覆盖、手动添加、置顶和删除。
- 会话附件：支持粘贴截图、拖拽或选择图片与常用文档；附件按会话本地存储，发送时进入 Codex 或 Responses 多模态输入。
- 家庭持仓：保留 `~/.trade-master/portfolio.json` 作为本人主账户，并在 `~/.trade-master/household/portfolio.json` 中管理家人和多个独立账户；每个账户分别记录持仓、成本、可用数量、风险偏好和监控开关，已确认成交会原子写入并生成审计 case。
- 关注列表：用户收藏与 Agent 候选标识，可真实添加、移除、运行可转债扫描，并按价格、涨幅、评分排序；选中标的后提供完整 K 线与技术指标，以及每 5 分钟结合实时行情、家庭持仓和纪律重新生成的 AI 买卖参考。页面不展示底层策略的原始信号或 K 线标记。
- 候选模型 V2：每 15 分钟枚举全部 A 股、ETF 和可转债，按资产类别独立初筛，再结合市场宽度、日线趋势、波动回撤与闭合 5/15 分钟结构输出最多 5 个关注候选；买入就绪候选允许为 0。模型表现按 1/3/5 日结果进入影子评估，在样本、胜率置信下界和回撤门槛全部满足前禁止标记为“已验证高置信”。详细口径见[候选模型 V2](docs/candidate-model-v2.md)。
- 决策闸门：数据、账户、纪律、成本、策略五层状态同步展示。
- 定时任务：从 manifest 安装盘前、盘中、收盘、盘后复盘和策略进化调度；支持启停、立即运行、固定会话与 JSON 审计。
- 场外 VOC：重点监控账号的公开微博、视频和文章经采集连接器进入独立事实域，按内容 ID 去重，提取板块、买卖表达、减仓/空仓/割肉/卖飞等仓位动作及动作后仓位，并形成当天与近 7 日的持仓、情绪和操作轨迹看板；所有动作与情绪要求原文证据，因子只参与风险加权，不能单独触发交易。连接器格式见[场外 VOC 采集接入](docs/voc-connector.md)。
- 通知：可在设置中配置并测试飞书只读通知；关键结果可推送给个人或群聊，并支持私聊机器人、群聊 @机器人对话；支持安装 SwiftBar 插件。飞书后台接入步骤见[飞书应用配置](docs/飞书应用配置.md)。
- AI Provider：OpenAI 兼容 API 或本机 Codex；API Key 使用 Electron 安全存储，不再提供伪造回复的演示 Provider。
- 应用更新：启动后静默检查、后台准备，下载完成后由用户重启安装；不覆盖 `~/.trade-master` 数据。
- Trade Master：通过 allowlist IPC 调用现有 Skill CLI，运行态仍写入 `~/.trade-master`。
- 策略实验室：自然语言生成 JSON 策略候选，支持验证升级、暂停、恢复与版本回滚，每次变更先保留快照。

## 技术架构

- Electron + electron-vite
- React 19 + TypeScript
- Node.js 20
- lucide-react
- JSON 事实源 + Markdown 审计报告

Renderer 不直接访问文件系统或启动进程。所有本地能力经 Preload 暴露的窄接口调用；Trade Master CLI 仅允许行情、分析、计划、关注池、复盘、通知、策略炼化等只读或域内命令，不开放任意 Shell。

AI 对话会在每次发送时读取最新 Trade Master 快照并注入持仓、关注池、目标、纪律、策略和进化规则；对话与消息保存在 `~/.trade-master/conversations/`。长期记忆单独保存在 `~/.trade-master/memory/memories.json`，只允许稳定偏好、长期目标、风险边界、习惯和经验，不能替代实时交易事实。数据缺失、行情失败或功能尚未安装时，界面只显示真实空状态和错误，不使用示例数据兜底。

## 本地运行

```bash
nvm use 20
npm install
npm run dev
```

## macOS 正式打包

面向普通用户分发前，必须完成 Developer ID 签名和 Apple 公证。配置证书与公证凭据后运行：

```bash
nvm use 20
npm run package:mac
```

完整配置见 [`docs/macos-distribution.md`](./docs/macos-distribution.md)。`package:mac:internal` 仅用于本机测试，不能发给外部用户。

跳过首次初始化直接查看工作台：

```text
http://localhost:5173/?skipOnboarding=1
```

## SwiftBar

将 [`swiftbar/韭菜盒子.5m.js`](./swiftbar/韭菜盒子.5m.js) 放入 SwiftBar Plugin Folder 并赋予执行权限。插件每 5 分钟读取 `~/.trade-master`，展示纪律、确认持仓和关注数量，只提供只读入口。

## 数据边界

- 持仓、成交、成本、现金和可用数量只认用户最新确认、本人 `portfolio.json` 或家庭账户事实仓；不同成员和账户禁止合并核算。
- 行情、策略信号、回测仓不能覆盖真实持仓。
- 任何建议买卖都必须带触发、失效、成本状态和下一检查点。
- `STOPPED` 只允许减风险、核账和复盘；收益目标不能放宽风险上限。
